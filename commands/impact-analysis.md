# /impact-analysis — 关联影响分析命令

源端系统变更时，拿变更清单 + 资产 knowledge，自动分析"源端的变化波及到本资产的哪些字段/表，
影响有多大"，输出影响清单 Excel。

## 定位

**影响清单 + 定位器，不是权威影响报告。** 核心价值 = 自动化两端，把中间留给人：
- 确定无影响 → 机器挡掉，不用看
- 确定有影响 → 机器圈出来，必须处理
- 判不了 → 诚实标"待确认"，需要人看代码

## 触发方式

```
/impact-analysis @变更清单.xlsx @资产knowledge目录
/impact-analysis 变更清单.xlsx DWB_TRADE_ORDER_D
"分析源端变更对 DWB_TRADE_ORDER_D 的影响"
```

### AI 判断规则

| 输入 | AI 判断 | 处理 |
|------|---------|------|
| `.xlsx` 变更清单 + knowledge 目录 | 完整流程 | 先跑 analyzer 产 knowledge（若无），再跑影响分析 |
| 变更清单 + 表名 | 先定位资产 | AI 在 `./output/` 或代码仓找 knowledge，找到后分析 |
| 只有变更清单 | 需要用户补充 | 提示用户指定要分析的资产 |

### 变更清单格式

模板在 `dws-pipeline-analyzer/templates/` 下：
- `变更清单_模板.xlsx` — 空模板（只有列头）
- `变更清单_示例.xlsx` — 带示例数据

三个 Sheet：
1. **表级变更**：切换前表名/切换后表名/是否平切/切换说明/表级变化类型
2. **字段级变更**：切换前后完整对照（库/schema/表/字段/类型）+ 字段变化类型
3. **变动类型说明**（可选）：类型 → 说明，用于报告翻译

**重要**：资产 SQL 引用的是**切换前表名**（旧名），工具据此匹配。

## 执行流程

### Step 1: 确保 knowledge 已生成（若没有则先跑 analyzer）

```bash
# 检查 knowledge 是否存在
ls {output_dir}/{资产英文名}/knowledge_draft.json

# 若不存在，先跑 analyzer（见 /analyze 命令）
python {skill_dir}/run.py analyzer --input {制品包或规则组目录} --output ./output/
```

### Step 2: 执行影响分析

```bash
python {skill_dir}/run.py impact_analyzer \
    --changes {变更清单.xlsx} \
    --knowledge {output_dir}/{资产英文名}/knowledge_draft.json \
    --output {output_dir}/{资产英文名}/impact.xlsx \
    --asset {资产英文名}
```

产出：`impact.xlsx`

### Step 3: 报告结果

向用户展示：

```
影响分析完成！

资产：{资产名}
源端变更：{字段变更数} 字段 / {表级变更数} 表

影响结果：
  🔴 有影响：{N} 项 — 必须处理
  🟡 待确认：{N} 项 — 需要人工判断
  🟢 无影响：{N} 项 — 已自动过滤
  ⚪ 未命中：{N} 项 — 与本资产无关

表级影响：{N} 项（平切/下线/初始化等）

报告：{output_dir}/{资产英文名}/impact.xlsx
  - 统计摘要 Sheet — 一眼看总览
  - 影响清单 Sheet — 🔴🟡 需关注的字段级影响
  - 表级影响 Sheet — 平切/下线/初始化等表级影响
  - 过滤摘要 Sheet — 🟢⚪ 无影响和未命中的完整记录
```

## 产出说明

### 统计摘要 Sheet
一眼看到被影响了几项、过滤了多少无关变更。

### 影响清单 Sheet（主表，只含 🔴🟡）
每行一个"字段被命中"的影响，列：
- **状态**：🔴有影响 / 🟡待确认
- **目标表/字段**：本资产受影响的字段（资产视角）
- **源表/字段**：哪个源端变更导致的
- **变化类型**：字段类型变化/字段下线/语义变化等
- **说明**：为什么判这个状态（cast吸收/截断/语义需确认等）
- **传播路径**：逐跳链路（规则/步骤:表达式）
- **涉及步骤/规则编码**：去代码里看哪里

### 表级影响 Sheet
平切/下线/初始化等**整表维度**的影响，列出该表在本资产的所有触点。

### 过滤摘要 Sheet（🟢⚪ 可追溯不吵）
机器判定无影响 + 与本资产无关的变更。信息不丢，但不在主表。

## 变化类型 → 影响判定逻辑

工具对每种变化类型有内置判定规则（映射表驱动）：

| 变化类型 | 默认状态 | 说明 |
|---------|---------|------|
| **字段级** | | |
| 字段类型及长度变化 | 动态 | 结合cast+DDL判断（吸收→🟢，截断→🔴，有cast→🟡） |
| 字段下线/删除 | 🔴 | 字段没了，取不到 |
| 新增字段 | ⚪ | 资产没引用 |
| 字段值语义变化 | 🟡 | 需语义判断，AI增强时可解读 |
| 字段名称变化 | 🟡 | 需改引用名 |
| 字段数据初始化（刷时间戳） | 动态 | 联动加载策略（全量🟢/增量🟡） |
| 字段数据初始化（不刷时间戳） | 动态 | 联动加载策略（全量🟢/增量🔴） |
| **表级** | | |
| 表/视图下线 | 🔴 | 来源消失，列出所有受波及字段 |
| 表/视图替换 | 🔴 | 需改来源表引用 |
| 表/视图主键变化 | 🔴 | 影响JOIN/去重 |
| 表/视图名称/schema变化 | 🟡 | 需改表名+术+规则+调度 |
| 平切 | 🟡 | 需替换表名引用 |
| 数据初始化（刷/不刷时间戳） | 动态 | 同字段级，联动加载策略 |
| 数据归档 | 🟡 | 需确认历史数据查询 |
| 取消权限 | 🔴 | 无法访问 |
| 数据硬删除 | 🔴 | 数据消失 |

## 关键规则

1. **第一步必须确保 knowledge 已生成**（先跑 /analyze）
2. **资产引用的是切换前表名**，工具据此匹配源变更
3. **平切表短路**：整表字段一致仅名称变化，不逐字段分析，直接归表级影响
4. **SELECT * 断链**：步骤用了通配无法枚举字段，诚实标🟡待确认并指出哪个步骤
5. **多源字段不综合判断**：一个字段被多个源命中时每源一行，综合判断交给读代码的人
6. **🟢无影响不进主表**，进过滤摘要 Sheet 可追溯，主表只留需关注的（🔴🟡）
7. **传播与判定分离**：传播是确定性图遍历，判定是规则映射，互不干扰

## SKILL 加载

此命令依赖 `dws-pipeline-analyzer` skill。在其他基于 opencode 的 agent 上使用时：
- 将 `.opencode/skills/dws-pipeline-analyzer/` 复制到目标项目
- 将此命令文件复制到 `.opencode/commands/`
- 确保 `run.py` 入口脚本与 `references/` 同级
- 模板文件在 `templates/` 下
