# /impact-analysis — 关联影响分析命令

源端系统变更时，拿变更清单分析"源端的变化波及到哪些字段/表，影响有多大"，输出影响清单 Excel。

支持两种模式，**根据输入自动分流**：

| 模式 | 触发条件 | 分析范围 |
|------|---------|---------|
| 单资产 | 用户指定了具体资产（knowledge 文件或表名） | 一个资产 |
| 跨资产 | 变更清单含 Sheet3（受影响表清单），用户给了代码仓路径 | 批量资产，自动定位 |

## 定位

**影响清单 + 定位器，不是权威影响报告。** 核心价值 = 自动化两端，把中间留给人：
- 确定无影响 → 机器挡掉，不用看
- 确定有影响 → 机器圈出来，必须处理
- 判不了 → 诚实标"待确认"，需要人看代码

## 触发方式

### 单资产模式

```
/impact-analysis @变更清单.xlsx @资产knowledge目录
/impact-analysis 变更清单.xlsx DWB_TRADE_ORDER_D
"分析源端变更对 DWB_TRADE_ORDER_D 的影响"
```

### 跨资产模式

```
/impact-analysis @变更清单.xlsx（含Sheet3受影响表清单） --repo-root 代码仓路径
"这批源端变更影响了哪些表，帮我批量分析"
```

### AI 判断规则

| 输入 | AI 判断 | 模式 | 脚本处理 |
|------|---------|------|---------|
| 变更清单 + knowledge 文件/目录 | 单资产 | 单资产 | 直接跑影响分析 |
| 变更清单 + 单个表名 | 先定位资产 | 单资产 | AI 找 knowledge 或代码仓目录 |
| 变更清单（含Sheet3）+ 代码仓路径 | 批量 | 跨资产 | `--cross-asset` 读 Sheet3 自动定位 |
| 变更清单（无Sheet3）+ 代码仓路径 | 需要用户补充 | — | 提示用户填 Sheet3 或指定资产 |

### 变更清单格式

模板在 `dws-pipeline-analyzer/templates/` 下：
- `变更清单_模板.xlsx` — 空模板（只有列头）
- `变更清单_示例.xlsx` — 带示例数据
- `跨资产案例.xlsx` — 跨资产场景案例

四个 Sheet：
1. **表级变更**：切换前表名/切换后表名/是否平切/切换说明/表级变化类型
2. **字段级变更**：切换前后完整对照（库/schema/表/字段/类型）+ 字段变化类型
3. **受影响表清单**：从血缘平台整理的受影响表名（跨资产模式用，一行一张表）
4. **变动类型说明**（可选）：类型 → 说明，用于报告翻译

**重要**：资产 SQL 引用的是**切换前表名**（旧名），工具据此匹配。

## 执行流程

> **⚠ 执行顺序铁律：直接执行脚本，不要先读变更清单。**
>
> 变更清单是结构化 Excel，脚本会自动解析。AI 不需要在执行前"理解"变更内容——
> 那既慢又消耗上下文（变更清单可能几百行），而且没有任何价值。
>
> 正确顺序：**拿到文件路径 → 直接执行脚本 → 看脚本输出 → 报告结果**。

---

### 单资产模式

#### Step 1: 确保 knowledge 已生成（若没有则先跑 analyzer）

```bash
# 若不存在，先跑 analyzer（见 /analyze 命令）
python {skill_dir}/run.py analyzer --input {制品包或规则组目录} --output ./output/
```

#### Step 2: 执行影响分析

```bash
python {skill_dir}/run.py impact_analyzer \
    --changes {变更清单.xlsx} \
    --knowledge {output_dir}/{资产英文名}/knowledge_draft.json \
    --output {output_dir}/{资产英文名}/impact.xlsx \
    --asset {资产英文名}
```

#### Step 3: 报告结果

```
影响分析完成！

资产：{资产名}
源端变更：{字段变更数} 字段 / {表级变更数} 表

影响结果：
  🔴 有影响：{N} 项 — 必须处理
  🟡 待确认：{N} 项 — 需要人工判断
  🟢 无影响：{N} 项 — 已自动过滤
  ⚪ 未命中：{N} 项 — 与本资产无关

表级影响：{N} 项（平切/下线/初始化等）

报告：{output_dir}/{资产英文名}/impact.xlsx
```

---

### 跨资产模式

#### Step 1: 执行跨资产影响分析

```bash
python {skill_dir}/run.py impact_analyzer --cross-asset \
    --changes {变更清单.xlsx} \
    --repo-root {代码仓根目录} \
    --output ./output/impact_cross_asset.xlsx
```

脚本自动完成：
1. 读 Sheet3 受影响表清单
2. 按表名在 `BFT/BftWideTable/` 下定位规则组目录（大小写不敏感、去 schema 前缀）
3. 每个资产自动跑 read_yml + analyze_pipeline + 影响分析
4. 汇总所有资产结果到一份 Excel

找不到目录的表会进"未处理" Sheet，不中断。

#### Step 2: 报告结果

```
跨资产影响分析完成！

受影响表：{N} 张
分析成功：{N} | 未定位：{N} | 出错：{N}

全链路影响合计：
  🔴 有影响：{N} 项
  🟡 待确认：{N} 项

报告：./output/impact_cross_asset.xlsx
  - 统计摘要 Sheet — 按资产分组统计 + 合计行
  - 影响清单 Sheet — 所有资产的 🔴🟡（有资产列区分）
  - 表级影响 Sheet — 所有资产的表级影响
  - 过滤摘要 Sheet — 所有资产的 🟢⚪
  - 未处理 Sheet — 未定位/出错的表
```

## 产出说明

### 单资产 impact.xlsx（四 Sheet）

| Sheet | 内容 |
|-------|------|
| 统计摘要 | 一眼看总览 |
| 影响清单 | 🔴🟡 字段级影响（主表，资产视角） |
| 表级影响 | 平切/下线/初始化等表级影响 |
| 过滤摘要 | 🟢⚪ 无影响和未命中（可追溯不吵） |

### 跨资产 impact_cross_asset.xlsx（五 Sheet）

| Sheet | 内容 |
|-------|------|
| 统计摘要 | 按资产分组统计 + 合计行 |
| 影响清单 | 所有资产的 🔴🟡（有"资产"列区分来源） |
| 表级影响 | 所有资产的表级影响（有"资产"列） |
| 过滤摘要 | 所有资产的 🟢⚪（有"资产"列） |
| 未处理 | 未定位到目录/分析出错的表 |

### 影响清单 Sheet 列说明

- **状态**：🔴有影响 / 🟡待确认
- **资产**（跨资产模式）：哪个资产受影响
- **目标表/字段**：受影响的字段（资产视角）
- **源表/字段**：哪个源端变更导致的
- **变化类型**：字段类型变化/字段下线/语义变化等
- **说明**：为什么判这个状态（cast吸收/截断/语义需确认等）
- **传播路径**：逐跳链路（规则/步骤:表达式）
- **涉及步骤/规则编码**：去代码里看哪里

## 变化类型 → 影响判定逻辑

工具对每种变化类型有内置判定规则（映射表驱动）：

| 变化类型 | 默认状态 | 说明 |
|---------|---------|------|
| **字段级** | | |
| 字段类型及长度变化 | 动态 | 兼容性链判定（源新类型 vs 目标DDL + cast关口） |
| 字段下线/删除 | 🔴 | 字段没了，取不到 |
| 新增字段 | ⚪ | 资产没引用 |
| 字段值语义变化 | 🟡 | 需语义判断，AI增强时可解读 |
| 字段名称变化 | 🟡 | 需改引用名 |
| 字段数据初始化（刷时间戳） | 动态 | 联动加载策略（全量🟢/增量🟡） |
| 字段数据初始化（不刷时间戳） | 动态 | 联动加载策略（全量🟢/增量🔴） |
| **表级** | | |
| 表/视图下线 | 🔴 | 来源消失，列出所有受波及字段 |
| 表/视图替换 | 🔴 | 需改来源表引用 |
| 表/视图主键变化 | 🔴 | 影响JOIN/去重 |
| 表/视图名称/schema变化 | 🟡 | 需改表名+术+规则+调度 |
| 平切 | 🟡 | 需替换表名引用 |
| 数据初始化（刷/不刷时间戳） | 动态 | 同字段级，联动加载策略 |
| 数据归档 | 🟡 | 需确认历史数据查询 |
| 取消权限 | 🔴 | 无法访问 |
| 数据硬删除 | 🔴 | 数据消失 |

## 关键规则

1. **直接执行脚本，不要先读变更清单**——脚本能自动解析 Excel，AI 预读既慢又无价值
2. **单资产需先有 knowledge**（先跑 /analyze）；跨资产自动跑 analyzer
3. **资产引用的是切换前表名**，工具据此匹配源变更
4. **跨资产表名定位**：表名归一化（去schema前缀+小写）匹配 `BFT/BftWideTable/` 下目录名，只扫 BftWideTable
5. **平切表短路**：整表字段一致仅名称变化，不逐字段分析，直接归表级影响
6. **SELECT * 断链**：步骤用了通配无法枚举字段，诚实标🟡待确认并指出哪个步骤
7. **多源字段不综合判断**：一个字段被多个源命中时每源一行，综合判断交给读代码的人
8. **🟢无影响不进主表**，进过滤摘要 Sheet 可追溯，主表只留需关注的（🔴🟡）
9. **传播与判定分离**：传播是确定性图遍历，判定是规则映射，互不干扰
