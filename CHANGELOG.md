# 更新日志

本文档记录 Analyzer Agent 每次版本更新的内容，方便大家了解新增了什么能力、修了什么问题。

---

## v1.4 — 2026-07-24

### 新功能
- **LTS 调度任务分析**：分析代码仓 yml 目录时，自动从 LTS 目录发现 F+I 调度任务，展示在资产说明书的"调度信息"区块
  - 按 V_GROUP_CODE 自动匹配规则组
  - 支持多个调度任务调同一规则组（临时/特殊场景）
  - 调度周期 cron 翻译成人话（`0 30 2 * * ?` → 每天 02:30）
  - 执行 Job 和调度依赖分开展示（依赖默认折叠）
  - F 任务显示依赖数量与来源表数量对比
  - 概览新增"调度类型"卡片（一天一调/一天多调）
- **LTS Excel 转 yml 工具**（`dev/lts_excel_to_yml.py`）：调度任务 Excel 批量转 yml

### 优化
- DDL 解析加表名索引 + 结果缓存，减少重复解析
- 调度信息 section 默认展开
- 技术设计文档新增第 10 章"调度信息"

### Bug 修复
- 修复 LTS yml 带 `*` 前缀字段名导致 YAML 解析失败（`*` 是 YAML alias 符号）
- 修复 block scalar 吞掉 Jobs 和 taskParams 子表
- 修复多参数提取只取到第一个的问题
- 修复字段详情 JOIN 关联同名字段重复显示
- 修复 field_usage_map 初始化不一致导致 KeyError（视图生成失败）
- 修复视图生成异常未被埋点记录

## v1.3 — 2026-07-23

### 新功能
- **运营埋点**：记录每次命令执行情况，用于运营统计
  - 本地 JSONL 存档 + 内网上报
  - 服务端看板（ECharts）：日活趋势/命令热度/用户活跃/资产 Top10/错误类型/完整分析链路
  - 分阶段耗时统计（解析/DDL发现/LTS发现/视图生成）
  - trace_id 关联 analyze → AI推理 → view-generator 完整链路
  - 半自动扩展机制（新字段/新命令零维护成本）

### 优化
- 数据库独立到家目录，代码更新不覆盖用户数据
- 服务端零依赖（Node 内置 node:sqlite，不需要编译）
- sync 脚本改用 git push 直推内网仓

### Bug 修复
- 修复 better-sqlite3 编译失败（改用 node:sqlite）
- 修复用户名 unknown（字段名 user vs user_name 不匹配）
- 修复统计次数重复（按 trace_id 去重）
- 修复 CSV 标题和列错位（换成 JSONL）

## v1.2 — 2026-07-22

### 新功能
- **多规则组链路分析（/analyze-chain）**：自动追溯整个代码仓的关联规则组，合并为一条从源表到最终资产的完整加工链路分析
  - 数据依赖递归追溯上游规则组
  - exec_sequence 按数据依赖拓扑排序
  - 排除 `_init` 规则组
  - 报告差异化（多规则组标识 + 规则组标签点击聚焦）

### 优化
- README 重写为小白友好版（快速开始 3 步上手）
- sync 脚本排除策略优化

## v1.1 — 2026-07-21

### 新功能
- **关联影响分析（/impact-analysis）**：源端变更 → 本资产受什么影响
  - 16 种变化类型，类型兼容性链判定
  - 单资产 + 跨资产批量模式
  - 4 Sheet Excel 输出（统计摘要/影响清单/表级影响/过滤摘要）

### 优化
- SQL AST 解析缓存 + DDL 目录扫描缓存
- 性能日志融入分析输出（慢阶段 >0.5s 自动显示）
- 字段使用检索（/field-search）支持多关键字

## v1.0 — 2026-07-14

### 初始版本
- **资产文档化（/analyze）**：分析一个 ETL 资产，生成字段映射 + 资产说明书 + 技术设计文档
  - 支持术加制品包 Excel 和代码仓 yml 两种输入
  - SQL 理解引擎：CTE/UNION/JOIN/窗口函数/注释别名/I视图封装
  - 自动检测加工模式/加载策略/类型兼容性/质量评估
  - 交互式 HTML 报告（数据流图/加工逻辑/字段映射/质量评估）
- **批量文档化（/analyze-batch）**：子进程隔离 + 单组隔离 + 同名去重
- 安装脚本（install.sh / install.bat）
